## 🧱 三、技术栈架构图

### 分层架构
```
┌─────────────────────────────────────────┐
│        API 层 (FastAPI)                 │
│  接口路由、参数校验、统一响应、限流      │
├─────────────────────────────────────────┤
│        Agent 编排层 (LangGraph)          │
│  状态管理、节点调度、并行执行、条件边    │
├─────────────────────────────────────────┤
│        服务层 (Service)                  │
│  LLM调用、搜索服务、API池、脱敏服务      │
├─────────────────────────────────────────┤
│        数据层 (Redis)                    │
│  缓存、任务状态持久化、限流计数          │
└─────────────────────────────────────────┘
```

### 核心组件关系
```
用户请求 → FastAPI接口 → 限流检查 → 缓存命中检查
                           ↓ 未命中
                    LangGraph Graph
                           ↓
            ┌──────────────┼──────────────┐
            ↓              ↓              ↓
      调度Agent      检索Agent(并行)  分析Agent(并行)
            ↓              ↓              ↓
            └──────────────┼──────────────┘
                           ↓
                    Schema校验Agent
                           ↓ 校验失败→重试
                    汇总Agent
                           ↓
                    Presidio脱敏
                           ↓
                    结果缓存 → 返回用户
```

## 💡 八、常见坑提前规避

1. **LangGraph版本**：必须用0.2.0+，旧版ParallelNode有bug
2. **DeepSeek API**：用`langchain-openai`兼容调用，不要自己封装HTTP
3. **Presidio模型**：第一次运行会自动下载模型，确保网络通畅
4. **Redis序列化**：State存Redis用pickle或json，注意类型转换
5. **异步陷阱**：LangGraph节点不要用CPU密集型操作，会阻塞事件循环
6. **API Key安全**：绝对不要硬编码在代码里，用.env文件，且加入.gitignore
