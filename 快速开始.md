# 快速开始

## 环境要求

- Python 3.11+
- Node.js 18+
- Redis（可选，用于缓存和任务持久化）

---

## 1. 后端启动

### 1.1 进入项目目录

```bash
cd deep_research_system
```

### 1.2 创建虚拟环境并安装依赖

```bash
python -m venv venv
source venv/bin/activate   # macOS/Linux
# venv\Scripts\activate    # Windows

pip install -r requirements.txt
# 如果没有 requirements.txt，使用 pyproject.toml：
pip install -e .
```

### 1.3 配置环境变量

```bash
cp .env.example .env
```

编辑 `.env`，至少填写以下必填项：

```ini
# 至少配置一个 LLM 的 API Key（四个槽位可共用同一个模型）
MODEL_REASONING_NAME=deepseek-chat
MODEL_REASONING_API_KEY=sk-your-key-here
MODEL_REASONING_API_BASE=https://api.deepseek.com

MODEL_SEARCH_NAME=deepseek-chat
MODEL_SEARCH_API_KEY=sk-your-key-here
MODEL_SEARCH_API_BASE=https://api.deepseek.com

MODEL_ANALYSIS_NAME=deepseek-chat
MODEL_ANALYSIS_API_KEY=sk-your-key-here
MODEL_ANALYSIS_API_BASE=https://api.deepseek.com

MODEL_WRITING_NAME=deepseek-chat
MODEL_WRITING_API_KEY=sk-your-key-here
MODEL_WRITING_API_BASE=https://api.deepseek.com

# 搜索服务（可选，不填则使用模拟数据）
TAVILY_API_KEY=tvly-your-key-here
```

> 最简配置：只需填写一个 `DEEPSEEK_API_KEY`，系统会自动回退使用它。

### 1.4 启动后端

```bash
# 方式一：直接运行
python main.py

# 方式二：uvicorn 命令
uvicorn main:app --reload --host 0.0.0.0 --port 8000
```

启动成功后：

| 地址 | 说明 |
|------|------|
| `http://localhost:8000` | API 服务 |
| `http://localhost:8000/docs` | Swagger 文档 |
| `http://localhost:8000/redoc` | ReDoc 文档 |
| `http://localhost:8000/api/health` | 健康检查 |

### 1.5 启动 Redis（可选）

```bash
# Docker 方式
docker run -d -p 6379:6379 --name agent-redis redis:7-alpine

# 或本地安装的 redis-server
redis-server
```

> 不启动 Redis 也能运行，系统会使用内存缓存作为回退。

---

## 2. 前端启动

### 2.1 进入前端目录

```bash
cd deep_research_system/frontend
```

### 2.2 安装依赖

```bash
npm install
```

### 2.3 启动开发服务器

```bash
npm run dev
```

启动成功后访问：`http://localhost:3000`

前端已配置代理，`/api` 请求会自动转发到后端 `http://localhost:8000`。

### 2.4 构建生产版本

```bash
npm run build
```

构建产物在 `frontend/dist/` 目录。

## 3. 快速验证

### 3.1 检查后端健康状态

```bash
curl http://localhost:8000/api/health
```

预期返回：

```json
{"code": 0, "msg": "ok", "data": {"status": "healthy"}}
```

### 3.2 查看模型配置

```bash
curl http://localhost:8000/api/models/status
```

### 3.3 发起一次研究任务

```bash
curl -X POST http://localhost:8000/api/research \
  -H "Content-Type: application/json" \
  -d '{"query": "2025年AI Agent市场趋势分析", "task_type": "industry_report", "depth": "quick"}'
```

### 3.4 运行测试

```bash
cd deep_research_system
python -m pytest tests/ -v
```

## 5. 常见问题

**Q: 启动报 `No module named 'xxx'`**
A: 确认在 `deep_research_system/` 目录下运行，且已激活虚拟环境。

**Q: 模型调用失败**
A: 检查 `.env` 中的 API Key 是否正确。访问 `http://localhost:8000/api/models/status` 查看模型状态。

**Q: 搜索没有结果**
A: 配置 `TAVILY_API_KEY`。未配置时系统使用模拟搜索数据。

**Q: 前端页面空白**
A: 确认后端已启动。前端 `/api` 请求通过 vite proxy 转发到后端。
