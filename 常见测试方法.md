⏺ 关于你的问题 — 这种场景下常见的测试方法：

  核心思路：Mock LLM Client，保留真实流程编排

  你的架构恰好适合这种模式 — 所有 agent 通过构造函数注入 LLMClient（依赖注入），所以只需要替换一个对象就能模拟整个 LLM 层。

  业界常见做法分几层：

  1. Mock LLM（最常用） — 用预制 JSON 响应替换 LLMClient.stream()。agent 的 _parse_output()
  会正常解析，拓扑的事件发射、状态流转、repair 循环都会真实运行。成本为零，覆盖最广。
  2. Snapshot 测试 — 记录一次真实 LLM 调用的完整输出，后续测试回放。适合回归测试。
  3. Contract 测试 — 只验证 API 请求/响应格式是否符合约定，不管内部逻辑。用 FastAPI 的 TestClient。
  4. 前端 store 测试 — 直接调用 store.addEvent() 传入模拟事件，验证 nodeStates、report 等状态变化。不需要后端。

  plan 里详细写了实施方案，核心是创建一个 MockLLMClient 和一组预制 JSON fixture，然后写拓扑流程测试。

 Mock 测试方案：不调用 LLM 验证前后端流程
  
 Context

 每次真实测试需要调用多个 LLM（planner、searcher、reader、analyzer、critic、writer、validator），成本高、耗时长。需要一套 mock
 测试方案，在不调用 LLM 的情况下验证：
 1. 后端拓扑流程是否正确执行（事件顺序、状态流转、repair 循环）
 2. 前端 store 是否正确处理各类事件
 3. SSE 事件流是否完整到达前端
 4. 前端组件渲染是否符合预期

 业界常见做法

 层级 1: 单元测试（Unit Tests）— 已有，需扩展

 - 测试纯函数、数据转换、配置解析
 - 不涉及 I/O，运行极快
 - 现有 32 个测试属于此类

 层级 2: Mock 集成测试（Mocked Integration Tests）— 本次重点

 - 替换 LLM 调用为预制的 JSON 响应
 - 验证 agent 解析、拓扑编排、事件发射的完整链路
 - pytest + unittest.mock.AsyncMock 或自定义 fixture
 - 运行快（秒级），覆盖核心逻辑

 层级 3: API/契约测试（Contract Tests）

 - 用 httpx.AsyncClient + FastAPI TestClient 测试 HTTP 端点
 - Mock service 层，验证请求/响应格式
 - SSE 端点通过消费 StreamingResponse 验证事件格式

 层级 4: 前端组件测试（Component Tests）

 - Vitest + @vue/test-utils 测试 Vue 组件
 - Mock Pinia store，验证渲染和交互
 - 不需要后端运行

 层级 5: E2E 测试（端到端）— 成本最高

 - Playwright/Cypress 驱动浏览器
 - Mock 后端 API 返回预制数据
 - 验证完整用户流程

 第一步：创建 MockLLMClient（核心基础设施）

 新建 tests/mock_llm.py，提供一个可配置的 mock LLM 客户端：

 class MockLLMClient:
     """根据 agent name 返回预制的 JSON 响应，模拟 LLM 输出。"""

     def __init__(self):
         self.responses: dict[str, str] = {}  # agent_name -> canned JSON
         self.call_log: list[dict] = []       # 记录每次调用的 prompt/agent

     def set_response(self, agent_name: str, json_data: dict):
         """为指定 agent 设置下次调用的返回值。"""
         self.responses[agent_name] = json.dumps(json_data, ensure_ascii=False)

     async def stream(self, model, prompt, system_prompt, api_key, result_collector=None):
         """模拟流式返回：将预制 JSON 逐 chunk 发送。"""
         # 记录调用
         self.call_log.append({"model": model.name, "prompt_len": len(prompt)})

         # 根据当前上下文确定返回哪个 agent 的响应
         content = self._match_response(prompt)

         # 模拟流式输出：每 100 字符一个 chunk
         for i in range(0, len(content), 100):
             yield content[i:i+100]

         if result_collector:
             result_collector.prompt_tokens = len(prompt) // 4
             result_collector.completion_tokens = len(content) // 4
             result_collector.latency_ms = 50.0

 关键设计：_match_response(prompt) 根据 prompt 内容（如包含的模板变量、agent
 特征词）自动匹配对应的预制响应。或者更简单：按调用顺序返回响应队列。
